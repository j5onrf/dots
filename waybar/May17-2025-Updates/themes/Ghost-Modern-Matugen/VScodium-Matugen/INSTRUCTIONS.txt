INSTRUCTIONS.txt - Theming VSCodium with Matugen for ML4W

-----------------------------------------------------------------------
Goal:
-----------------------------------------------------------------------
These instructions will guide you on how to use Matugen to dynamically
theme VSCodium (or VS Code) based on your current wallpaper. The
included `vscodium-matugen.sh` script parses colors generated by Matugen
and applies them to VSCodium's user interface.

This setup assumes you are using Matugen to generate a CSS file
containing color definitions (e.g., for Waybar or another component),
which this script will then read.

-----------------------------------------------------------------------
Prerequisites:
-----------------------------------------------------------------------
1.  **Matugen Installed and Configured:**
    *   You must have Matugen installed (e.g., via `cargo install matugen`).
    *   Your Matugen setup should be configured to generate a CSS color
        definition file when your wallpaper changes. The provided
        `vscodium-matugen.sh` script is set up to parse a file in the
        `@define-color color_name #RRGGBB;` format.
    *   Refer to the example Matugen `config.toml` in this repository to see
        how `~/.config/waybar/colors.css` is generated from a template.
        You will need a similar setup.

2.  **VSCodium (or VS Code) Installed.**

3.  **`jq` Installed:** A command-line JSON processor.
    *   On Arch Linux: `sudo pacman -S jq`
    *   Other distros: Use your package manager (e.g., `sudo apt install jq`).

4.  **`awk` Installed:** A powerful text processing tool.
    *   This is usually installed by default on most Linux systems.

5.  **A Wallpaper Changer Script:** You need a script that changes your
    wallpaper and then triggers Matugen (e.g., `wallpaper.sh` if you
    are using a setup like ml4w).

-----------------------------------------------------------------------
Setup Steps:
-----------------------------------------------------------------------
1.  **Place the `vscodium-matugen.sh` Script:**
    *   Copy the `vscodium-matugen.sh` script (from this repository) to a
        suitable location on your system, for example:
        `~/.config/hypr/scripts/` (if using Hyprland like the original setup)
        or `~/.local/bin/`
    *   Make the script executable:
        `chmod +x /path/to/your/vscodium-matugen.sh`

2.  **Configure `vscodium-matugen.sh`:**
    *   Open the `vscodium-matugen.sh` script in a text editor.
    *   **IMPORTANT:** Modify the `CSS_FILE_TO_PARSE` variable (near the top)
        to point to the ACTUAL path of the CSS file Matugen generates with
        your theme's colors. In the original setup, this was
        `"$HOME/.config/waybar/colors.css"`. Yours might be different based
        on your Matugen `config.toml`.
        ```bash
        CSS_FILE_TO_PARSE="$HOME/.config/your/path/to/matugen-colors.css"
        ```
    *   Verify the `VSCODIUM_SETTINGS_FILE` variable. The default is for
        VSCodium on Linux (`"$HOME/.config/VSCodium/User/settings.json"`).
        If you use VS Code, or a Flatpak version, adjust this path.
        Examples are commented in the script.

3.  **Integrate with Your Wallpaper Changer Script:**
    *   Edit the script you use to change wallpapers and apply Matugen themes
        (e.g., `wallpaper.sh`).
    *   You need to call `vscodium-matugen.sh` *AFTER* Matugen has
        successfully run and generated its color files.
    *   Add a section similar to this in your wallpaper script, adjusting the
        path to `vscodium-matugen.sh` if needed:

        ```bash
        # ... (your script lines that execute Matugen, e.g., matugen image "$wallpaper") ...
        echo ":: Matugen execution finished." # Example log

        # --- Apply Matugen theme to VSCodium ---
        if [ -f "/path/to/your/vscodium-matugen.sh" ]; then
            echo ":: Applying Matugen theme to VSCodium"
            bash "/path/to/your/vscodium-matugen.sh"
        else
            echo ":: VSCodium Matugen script not found, skipping."
        fi
        # --- End of VSCodium section ---

        # ... (rest of your wallpaper script, e.g., reloading Waybar) ...
        ```

4.  **Initial Test:**
    *   Manually run your main wallpaper changing script to select a new wallpaper.
    *   Observe any output for messages like "Applying Matugen theme to VSCodium".
    *   Alternatively, for a direct test, run `vscodium-matugen.sh` manually:
        `bash /path/to/your/vscodium-matugen.sh`
    *   Close VSCodium completely (pkill if necessary) and restart it.
    *   The theme colors should have changed! The script creates a backup of
        your VSCodium `settings.json` before modifying it.

-----------------------------------------------------------------------
Customizing VSCodium's Appearance:
-----------------------------------------------------------------------
The `vscodium-matugen.sh` script contains a large block of JSON assigned
to the `CUSTOMIZATIONS_JSON` variable (using a "heredoc"). This block
defines how Matugen's extracted colors are mapped to VSCodium's UI elements.

1.  **Identify Matugen Color Variables:**
    *   At the top of `vscodium-matugen.sh`, various Bash variables (e.g.,
        `PRIMARY`, `SURFACE_CONTAINER`, `ON_BACKGROUND`) are assigned hex codes
        extracted from your Matugen CSS file. The script includes comprehensive
        debug output to show you which colors are being extracted.

2.  **Edit the `CUSTOMIZATIONS_JSON` Block:**
    *   Find the VSCodium UI key you want to change (e.g.,
        `"activityBar.background"`).
    *   Change its value to use a different Matugen color variable from the
        extraction section (e.g., `"#${SURFACE_BRIGHT}"` or
        `"#${PRIMARY_CONTAINER:-${SURFACE_BRIGHT}}" ` for a fallback).
    *   Refer to the official VS Code Theme Color reference for a list of
        all available VSCodium UI keys:
        https://code.visualstudio.com/api/references/theme-color

3.  **Test Your Changes:**
    *   Save `vscodium-matugen.sh`.
    *   Run it manually: `bash /path/to/your/vscodium-matugen.sh`
    *   Reload VSCodium (Ctrl+Shift+P -> "Reload Window").
    *   Repeat until you achieve your desired look.

-----------------------------------------------------------------------
Troubleshooting:
-----------------------------------------------------------------------
*   **Colors Not Changing:**
    1.  Ensure your wallpaper script is actually calling `vscodium-matugen.sh`.
        Add `echo` statements in both scripts to trace execution.
    2.  Run `vscodium-matugen.sh` manually. Check for any error messages.
    3.  Verify `CSS_FILE_TO_PARSE` path is correct and the file exists and
        contains `@define-color` definitions.
    4.  Verify `VSCODIUM_SETTINGS_FILE` path is correct.
    5.  Check the "COMPREHENSIVE DEBUG" output from the script. Are color
        variables empty? If so, the color name in `get_color_from_css "name"`
        might be misspelled or not present in your Matugen CSS output.
    6.  If the script reports "CUSTOMIZATIONS_JSON was empty or invalid", copy
        the "Problematic JSON content" it prints into an online JSON validator
        to find the syntax error. This usually means a Bash variable expanded
        to an empty string within the JSON, resulting in something like `"#",`.
    7.  Manually inspect your VSCodium `settings.json` to see if the
        `workbench.colorCustomizations` block was written or if it's malformed.

*   **`awk` or `jq` errors:** Ensure these tools are installed correctly.

-----------------------------------------------------------------------
Important Note on CSS Format:
-----------------------------------------------------------------------
The provided `vscodium-matugen.sh` script uses `awk` in its
`get_color_from_css` function to parse lines like:
`@define-color background #111318;`

If your Matugen setup generates CSS custom properties instead (e.g.,
`--background: #111318;`), you will need to modify the
`get_color_from_css` function in `vscodium-matugen.sh` to correctly
parse that format (e.g., using `grep` and `sed`, or a different `awk` pattern).
